# FitnessClub Agent

Агент клуба: **QR → CRM (клиент + абонемент) → открытие турникета C01**.  
Все HTTP-запросы к CRM и команды на C01 пишутся в **Журнал**.

## EXE

`dist\FitnessClubAgent.exe` — закройте старую копию перед пересборкой `.\build.ps1`.

## Быстрый старт

1. **CRM** — в поле URL укажите **только корень сайта**, без `/admin` и без `/api/v1`  
   (пример: `https://worldcashfit.ru`). Иначе heartbeat уйдёт на `.../admin/api/v1/...` и вернётся **404**.  
   Дальше `gateway_token` → **Сохранить всё** → **Проверить CRM** (в журнале `HTTP 200`).
2. **Оборудование** — режим **Слушать**, host `0.0.0.0`, порт `8765`, `net.server` = IP ПК.
3. **Запустить агент** — C01 **[ON]** (или **симулятор** без железа).
4. **Проход** — QR с телефона (< 15 сек) → **Проверить проход** — смотрите **Журнал**.

## Что в журнале при скане

```
══════ ПРОХОД: скан QR ══════
← C01 event card | id=FITNESSCLUB:ENTRY:...
────── CRM запрос ──────
→ POST /api/v1/gateway/access/entry
  тело: {"qr":"...", "device_id":"..."}
← HTTP 200
  тело: {"access_granted":true, "user":{...}}
  итог: ДОПУСК — Иван Иванов
────── C01 команда (открыть турникет) ──────
→ {"control":"exdev", ...}
```

При отказе: `access_granted:false`, `reason=...`, команда **запрет прохода**.

## HTTP 401 на heartbeat

Тело `{"ok":false,"reason":"unauthorized"}` — в CRM **нет клуба** с таким `gateway_token`. Не путать с логином пользователя приложения: нужен именно токен из карточки клуба в админке. После ротации токена в CRM обновите его в агенте.
